Setup Fastlane

1. Install rbenv: brew install rbenv // If not installed.

2. Create .ruby-version file and add version "4.0.2".

3. Install bundler: gem install bundler // If not installed.

4. Create Gemfile and add below:

a. Run Keychain Access

b. Select 'login' from left side

c. Menu 'Certificate Assistant -> Request a Certificate From a Certificate Authority'

d. Input email, select 'Saved to disk'

e. With that file, create a distribution certificate from Apple

f. Download the distribution certificate

g. Create a distribution profile, and download that

2. Double click the downloaded 'distribution.cer', step 1.f

3. In Keychain Access, find the distribution certificate, "My Certificates" tab

4. Export that as '.p12' file. While exporting, enter password. This is P12_PASSWORD secret in GitHub

5. Move to '.p12' file and run: base64 -i that_file.p12 | pbcopy

6. Add that value to GitHub secret as BUILD_CERTIFICATE_BASE64

7. Move to distribution profile file and run: base64 -i provision_file.mobileprovision | pbcopy

8. Add that value to GitHub secret as BUILD_PROVISION_PROFILE_BASE64

GitHub secret variables

* BUILD_CERTIFICATE_BASE64: Access Certificate. To prove user.

base64 -i BUILD_CERTIFICATE.p12 | pbcopy

* P12_PASSWORD: Protect Access Certificate password

* BUILD_PROVISION_PROFILE_BASE64: Provision. To allow what to do.

base64 -i PROVISIONING_PROFILE.mobileprovision | pbcopy

* KEYCHAIN_PASSWORD: Temporary keychain password on runner. Any password.

* APP_STORE_CONNECT_KEY_ID: App Store Connect key. For distribution or TestFlight

a. Go to App Store Connect, Users and Access tab

b. Left side: Keys -> App Store Connect API

c. For CI, use Team Keys

d. Copy 'KEY ID'

* APP_STORE_CONNECT_ISSUER_ID: App Store Connect user

e. Copy 'Issuer ID'

* APP_STORE_CONNECT_PRIVATE_KEY: App store Connect private key

f. Download AuthKey_********.p8 file

g. Run: base64 -i AuthKey_********.p8 | pbcopy

* DISTRIBUTION_PROVISION_NAME: Distribution provision profile name. My custom variable

The name in "Certificates, Identifiers & Profiles" site -> Profiles menu

Xcode settings: Signing & Capabilities

When uploading the archive to TestFlight or AppStore, it only use the DISTRIBUTION certificate and provisioning profile. So you should set the Signing & Capabilities of Release manually. Otherwise git runner will print this message "... Xcode couldn't find any iOS App Development provisioning profiles ...".

Running KMP iOS prject with Xcode

1. Have to use Java 17, not 25 (latest version)

2. Install Java using brew: brew install openjdk@17

3. Run this command in terminal.

sudo ln -sfn "$(brew --prefix openjdk@17)/libexec/openjdk.jdk" /Library/Java/JavaVirtualMachines/openjdk-17.jdk /usr/libexec/java_home -V

4. Add this to ~/.zshrc

export JAVA_HOME=$(/usr/libexec/java_home -v 17)

export PATH="$JAVA_HOME/bin:$PATH"

5. For debuging, add this script to Xcode build phase.

export JAVA_HOME="$(/usr/libexec/java_home -v 17)"

export PATH="$JAVA_HOME/bin:$PATH"

echo "JAVA_HOME=$JAVA_HOME"

java -version
